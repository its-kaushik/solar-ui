## 1. Executive Summary

This document outlines the requirements for the company's official website — the primary digital presence for attracting customers, educating them about solar benefits and subsidies, and capturing leads.

## 4. Subsidy Information (Key Website Content)

### 4.1 Central Government — PM Surya Ghar Muft Bijli Yojana

| System Size | Subsidy Amount |
|-------------|---------------|
| 1 KW | Rs 30,000 |
| 2 KW | Rs 60,000 |
| 3 KW and above | Rs 78,000 (capped) |

- Subsidy disbursed via Direct Benefit Transfer (DBT) to customer's bank account.
- Available only for residential consumers.
- System must be on-grid with net metering.

### 4.2 Delhi State Government — Additional Subsidy

| Benefit | Details |
|---------|---------|
| **State capital subsidy** | Rs 2,000/kW (up to Rs 10,000 per consumer) |
| **Additional state subsidy** | Rs 30,000 for 3 KW rooftop solar installations |
| **Combined max (Central + State) for 3 KW** | Up to Rs 1,08,000 |

### 4.3 Delhi Generation-Based Incentive (GBI)

| System Size | GBI Rate | Duration |
|-------------|----------|----------|
| Up to 3 KW | Rs 3/kWh | 5 years |
| 3 KW to 10 KW | Rs 2/kWh | 5 years |

- GBI is first adjusted against the monthly electricity bill; surplus is deposited into the customer's bank account.
- Available to domestic, commercial, and industrial consumers.

### 4.4 Other Delhi Benefits

- **Free bi-directional meter** from DISCOM (systems up to 10 KW)
- **100% exemption** from wheeling charge, banking charge, and cross-subsidy surcharge (systems commissioned before March 2027)
- **Net metering application fees waived**
- **Accelerated depreciation** benefits for commercial/industrial consumers
- **Solar plants up to 500 KVA** exempt from electrical inspection

### 4.5 Combined Savings Example (3 KW Residential System)

This should be presented as a clear infographic/card on the website:

| Benefit | Value |
|---------|-------|
| Central subsidy | Rs 78,000 (one-time) |
| Delhi state subsidy | Up to Rs 30,000 (one-time) |
| GBI income (Rs 3/kWh x 5 years) | ~Rs 50,000/year |
| Electricity bill savings | ~Rs 50,000/year |
| Free net meter | Included |

**Key message:** With subsidies covering a significant portion of the cost and ongoing GBI + bill savings, the system pays for itself in approximately 3-4 years, then generates free electricity for 20+ more years.

English only (for now). Structure should allow for Hindi addition in the future if needed.

### 5.3 Design Principles

- **Mobile-first (non-negotiable)** — The website is designed for mobile screens first, then enhanced for tablets and desktops. Our target audience in Delhi primarily uses smartphones — they'll find us via Google search on mobile, receive our link via WhatsApp, or tap our ad on social media. Every component, layout, and interaction must work flawlessly on a 375px-wide screen before any desktop styling is considered. This includes:
  - All buttons and links must be large enough to tap (minimum 44×44px touch targets)
  - Forms must use mobile-native inputs (numeric keypad for phone, etc.)
  - Tables must be scrollable on small screens
  - No content should be hidden behind hover-only interactions
  - Sticky call/WhatsApp bar on mobile for instant contact
- **Fast loading** — Static generation wherever possible; optimize images (WebP/AVIF). Mobile users are often on 4G/variable networks — page load must be under 2 seconds
- **Trust-forward** — Display credentials, registrations, and subsidy info prominently
- **Action-oriented** — Every page should have a clear CTA (contact form, WhatsApp, call). On mobile, a sticky bottom bar with Call and WhatsApp buttons should be visible on key pages
- **SEO-optimized** — Proper meta tags, structured data, semantic HTML, fast Core Web Vitals

#### 6.2.7 Solar Calculator

**Purpose:** Interactive tool to engage visitors and generate qualified leads.

**Inputs:**
- Monthly electricity consumption (in kWh/units)
- Type of property: Residential / Commercial
- Area (optional): South Delhi / Faridabad / Gurgaon — to determine subsidy eligibility

**Outputs (calculated client-side):**
- Recommended system size (KW)
- Estimated system cost (indicative range, since pricing is quote-based)
- Applicable central subsidy amount
- Applicable Delhi state subsidy amount (if eligible)
- Estimated GBI income per year (if eligible)
- Net cost after subsidies
- Estimated monthly electricity bill savings
- Estimated payback period (years)
- 25-year total savings

**Calculation Logic (approximate):**
- 1 KW generates ~4 units/day = ~120 units/month (Delhi average)
- System size = monthly consumption / 120 (rounded up)
- Cost per KW = indicative range (to be defined, e.g., Rs 55,000 - 65,000/KW before subsidy)
- Subsidy as per slabs
- Savings = units generated x per-unit electricity rate (BSES slab rates)

**Commercial Threshold Logic:**
- If property type is "Commercial" **and** the recommended system size exceeds 15 KW, the calculator should **not** display a cost estimate.
- Instead, show: "For commercial systems above 15 KW, pricing is customized based on site conditions. Contact us for a free assessment."
- Rationale: Residential and small commercial systems have relatively standard per-KW pricing, but large commercial installations vary significantly based on roof type, structure requirements, electrical infrastructure, and project complexity. Showing a flat per-KW estimate for a 50 KW system would be misleading.
- For commercial systems up to 15 KW, the calculator can show estimates with the same logic as residential (but without central subsidy, since PM Surya Ghar is residential-only).

**Post-calculation CTA:**
- "Get an Exact Quote" → pre-filled contact form or WhatsApp message with calculated details

**Note:** Show a disclaimer that these are estimates and actual figures may vary based on site conditions, roof orientation, shading, etc.

## 7. Technical Architecture

### 7.1 Tech Stack

| Layer | Technology | Justification |
|-------|-----------|---------------|
| **Framework** | Next.js 15 (App Router) | SSG/SSR for SEO, React for interactivity, API routes for form handling |
| **Styling** | Tailwind CSS v4 | Rapid development, responsive design, consistent design system |
| **Blog** | MDX (next-mdx-remote or @next/mdx) | Markdown-based blog posts with React component support, managed via code |
| **Contact Form** | Next.js API Route + email service + Google Sheets | Email notification + Google Sheet as lightweight CRM |
| **Email Service** | Resend or Nodemailer | Transactional email for contact form submissions |
| **Icons** | Lucide React | Clean, consistent icon set |
| **Animations** | Framer Motion (optional) | Subtle animations for engagement |
| **Deployment** | Vercel (recommended) or self-hosted | Vercel offers free tier with great Next.js optimization; alternatively deploy on user's server |

### 7.2 Architecture Decisions

- **No database** — All content is static (MDX files, hardcoded data). Contact form submissions are emailed directly. No user accounts.
- **No CMS / admin panel** — Blog posts managed via MDX files in the repository. New posts require a code commit and deployment.
- **Static Site Generation (SSG)** — All pages are pre-rendered at build time for maximum speed and SEO. Only the contact form API route runs server-side.
- **No authentication** — No login, no user accounts, no protected routes.

### 7.4 Performance Targets

| Metric | Target |
|--------|--------|
| Lighthouse Performance | 90+ |
| Lighthouse SEO | 95+ |
| Lighthouse Accessibility | 95+ |
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Total page size | < 500 KB (initial load) |

**Accessibility (a11y) Requirements:**
- WCAG 2.1 AA compliance as a baseline
- Proper color contrast ratios (especially important given the navy/amber/green palette — verify all text-on-background combinations meet 4.5:1 for normal text, 3:1 for large text)
- All images must have descriptive `alt` text
- ARIA labels on interactive elements (form fields, buttons, calculator inputs)
- Keyboard navigability for all interactive components (contact form, solar calculator, mobile menu)
- Semantic HTML throughout (proper heading hierarchy, landmark regions, lists)
- Good accessibility also contributes positively to Google's SEO ranking signals

### 7.5 Responsive Breakpoints (Mobile-First)

All CSS is written with mobile as the **default** — desktop styles are added via progressive enhancement (`sm:`, `md:`, `lg:` prefixes). This is the opposite of designing for desktop and then "making it fit" on mobile.

| Breakpoint | Width | Target | Priority |
|------------|-------|--------|----------|
| Base (no prefix) | 0 - 639px | Mobile phones | **Primary — designed first** |
| `sm:` | 640px+ | Large phones / small tablets | Secondary |
| `md:` | 768px+ | Tablets | Secondary |
| `lg:` | 1024px+ | Laptops / desktops | Enhancement |
| `xl:` | 1280px+ | Large desktops | Enhancement |

**Primary test device:** 375px width (iPhone SE size). If it works here, it works everywhere.
